# CSS 文件拆分方案

## 概述

为了提高代码的可维护性和模块化程度，我们将原来的单一 `styles.css` 文件拆分为多个功能模块化的 CSS 文件。

## 拆分方案

### 1. `base.css` - 基础样式文件
- **功能**: 包含工具类、动画效果和基础样式定义
- **内容**:
  - Tailwind CSS 工具类扩展
  - 自定义工具类（text-shadow, bg-stars, glass-effect 等）
  - 动画效果（twinkle, progress-animation, pulse）
  - 文件类型图标颜色
  - 基础布局样式
  - 全局样式（模态框打开时的 body 样式）

### 2. `scrollbar.css` - 滚动条样式文件
- **功能**: 包含所有滚动条相关的样式
- **内容**:
  - 主题滚动条样式
  - 预览内容滚动条样式
  - 全局滚动条样式
  - Firefox 滚动条样式
  - 文件列表滚动条样式
  - 模态框内滚动条样式

### 3. `breadcrumb.css` - 面包屑导航样式文件
- **功能**: 包含面包屑容器、按钮和文件夹名称样式
- **内容**:
  - 面包屑导航容器样式
  - 面包屑返回按钮样式
  - 可展开文件夹样式
  - 子分类展开样式
  - 子文件类型按钮样式
  - 按钮动画效果

### 4. `file-filters.css` - 文件类型过滤器样式文件
- **功能**: 包含文件类型按钮、过滤器容器等样式
- **内容**:
  - 文件类型过滤器容器
  - 文件类型按钮基础样式
  - 不同文件类型的颜色变体
  - 响应式设计

### 5. `preview.css` - 文件预览样式文件
- **功能**: 包含预览模态框、全屏模式、预览内容等样式
- **内容**:
  - 全屏模式样式
  - 预览模态框基础样式
  - 预览内容样式
  - 预览图片/视频/音频容器
  - 文本内容样式（prose）
  - 全屏文本和 Markdown 样式

### 6. `file-grid.css` - 文件网格样式文件
- **功能**: 包含文件卡片、文件夹卡片、网格布局等样式
- **内容**:
  - 文件网格布局
  - 文件卡片样式
  - 文件夹网格布局
  - 文件夹卡片样式
  - 响应式网格布局
  - 文件信息样式
  - 模态框背景样式

### 7. `notifications.css` - 通知和消息框样式文件
- **功能**: 包含全局消息框、通知容器等样式
- **内容**:
  - 全局消息框样式
  - 全局通知容器样式
  - 动画效果（slideIn, progress）
  - 响应式设计

## 加载机制

### CSS 加载器 (`css-loader.js`)
- **功能**: 动态加载拆分后的 CSS 文件
- **特性**:
  - 自动加载所有 CSS 文件
  - 防止重复加载
  - 添加版本号防止缓存
  - 错误处理和日志记录
  - 支持加载特定 CSS 文件

### 使用方法

1. **自动加载**: 页面加载时自动加载所有 CSS 文件
```javascript
// 页面加载完成后自动执行
window.cssLoader.loadAllCSS();
```

2. **手动加载特定文件**:
```javascript
// 加载特定的 CSS 文件
await window.cssLoader.loadSpecificCSS('base.css');
```

3. **检查加载状态**:
```javascript
// 检查文件是否已加载
const isLoaded = window.cssLoader.isLoaded('/static/css/base.css');

// 获取已加载的文件列表
const loadedFiles = window.cssLoader.getLoadedFiles();
```

## 优势

1. **模块化**: 每个 CSS 文件专注于特定功能
2. **可维护性**: 更容易定位和修改特定样式
3. **按需加载**: 可以根据需要加载特定的样式文件
4. **缓存优化**: 独立的文件可以更好地利用浏览器缓存
5. **团队协作**: 不同开发者可以并行处理不同的样式文件

## 文件结构

```
front/css/
├── base.css              # 基础样式
├── scrollbar.css         # 滚动条样式
├── breadcrumb.css        # 面包屑导航样式
├── file-filters.css      # 文件类型过滤器样式
├── preview.css           # 文件预览样式
├── file-grid.css         # 文件网格样式
├── notifications.css     # 通知和消息框样式
├── custom.css            # 自定义样式（保留）
├── styles.css            # 原始样式文件（保留作为备份）
└── README.md            # 说明文档
```

## 注意事项

1. **兼容性**: 确保所有拆分后的样式与原始样式保持一致
2. **依赖关系**: 注意 CSS 文件之间的依赖关系，确保加载顺序正确
3. **性能**: 虽然文件被拆分，但通过 CSS 加载器的优化，不会影响页面加载性能
4. **调试**: 在开发过程中，可以通过浏览器开发者工具查看每个 CSS 文件的加载状态
