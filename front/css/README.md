# CSS 样式文件拆分说明

## 拆分概述

将原来的 `styles.css` 文件按照功能模块拆分为多个独立的样式文件，提高代码的可维护性和可读性。

## 文件结构

```
front/css/
├── styles.css          # 主样式文件（引入所有模块）
├── utilities.css       # 基础工具类样式
├── scrollbar.css       # 滚动条样式
├── breadcrumb.css      # 面包屑导航样式
├── file-filters.css    # 文件类型过滤器样式
├── preview-modal.css   # 预览模态框样式
├── file-cards.css      # 文件卡片样式
├── notifications.css   # 通知系统样式
├── responsive.css      # 响应式布局样式
└── README.md          # 说明文档
```

## 模块说明

### 1. utilities.css - 基础工具类样式
- **功能**: 通用工具类、动画效果、基础样式
- **包含内容**:
  - `@layer utilities` 工具类
  - 自定义样式类（`.text-shadow`, `.bg-stars`, `.glass-effect` 等）
  - 文件类型图标颜色
  - 进度条和脉冲动画
  - 全屏模式样式
  - 禁用状态样式

### 2. scrollbar.css - 滚动条样式
- **功能**: 所有滚动条相关的样式
- **包含内容**:
  - 主题滚动条样式
  - 预览内容滚动条样式
  - 全局滚动条样式
  - Firefox 滚动条样式
  - 特殊滚动条样式（文件列表、模态框等）

### 3. breadcrumb.css - 面包屑导航样式
- **功能**: 面包屑导航相关的样式
- **包含内容**:
  - 面包屑容器样式
  - 返回按钮样式
  - 文件夹名称样式

### 4. file-filters.css - 文件类型过滤器样式
- **功能**: 文件类型过滤器和标签相关的样式
- **包含内容**:
  - 可展开标签样式
  - 子分类展开容器样式
  - 子分类按钮样式
  - 按钮进入动画

### 5. preview-modal.css - 预览模态框样式
- **功能**: 文件预览相关的所有样式
- **包含内容**:
  - 预览模态框样式
  - 模态框滚动条控制
  - 预览文件信息样式
  - 预览关闭按钮样式
  - 视频、音频、图片预览优化
  - Markdown 预览样式
  - 全屏预览样式
  - 遮罩层样式

### 6. file-cards.css - 文件卡片样式
- **功能**: 文件和文件夹卡片的网格布局和样式
- **包含内容**:
  - 文件卡片网格布局
  - 文件夹网格布局
  - 响应式布局优化
  - 卡片内容对齐样式
  - 文件信息文本对齐
  - 网格列跨度类
  - 外站文档特殊样式
  - 文件图标容器优化

### 7. notifications.css - 通知系统样式
- **功能**: 消息框和通知相关的样式
- **包含内容**:
  - MessageBox 样式（顶部通知）
  - Notify 样式（右上角通知）
  - 边框发光效果
  - 炫光效果
  - 移动端适配

### 8. responsive.css - 响应式布局样式
- **功能**: 响应式设计和媒体查询
- **包含内容**:
  - 响应式布局优化
  - 移动端通知样式

## 使用方法

主样式文件 `styles.css` 通过 `@import` 语句引入所有拆分的样式模块：

```css
/* 引入基础工具类样式 */
@import url('./utilities.css');

/* 引入滚动条样式 */
@import url('./scrollbar.css');

/* 引入面包屑导航样式 */
@import url('./breadcrumb.css');

/* 引入文件类型过滤器样式 */
@import url('./file-filters.css');

/* 引入预览模态框样式 */
@import url('./preview-modal.css');

/* 引入文件卡片样式 */
@import url('./file-cards.css');

/* 引入通知系统样式 */
@import url('./notifications.css');

/* 引入响应式布局样式 */
@import url('./responsive.css');
```

## 优势

1. **模块化**: 按功能模块拆分，便于维护和修改
2. **可读性**: 每个文件职责单一，代码结构清晰
3. **可扩展性**: 新增功能时只需修改对应的模块文件
4. **团队协作**: 不同开发者可以并行修改不同模块
5. **性能优化**: 可以按需加载特定的样式模块

## 注意事项

1. 所有拆分的样式文件都包含在 `front/css/` 目录下
2. 主样式文件 `styles.css` 通过相对路径引入其他模块
3. 每个模块文件都有详细的注释说明其功能
4. 拆分过程中确保没有遗漏或重复的样式规则
5. 保持原有的样式效果和功能不变

## 验证

拆分完成后，需要验证：
- 所有样式正常加载
- 页面显示效果与原文件一致
- 所有功能正常工作
- 响应式布局正常
- 动画效果正常
