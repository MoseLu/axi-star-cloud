# 宝塔面板配置指南

## 问题描述
生产环境无法访问 `/uploads/` 路径下的文件，返回404错误。

## 解决方案

### 1. 检查项目部署

在宝塔面板终端中执行以下命令：

```bash
# 进入项目目录
cd /www/wwwroot/axi-star-cloud

# 拉取最新代码
git pull origin main

# 创建uploads目录结构
mkdir -p uploads/{image,avatars,video,audio,document,md,other}

# 设置权限
chmod -R 755 uploads
chown -R www:www uploads

# 创建测试文件
echo "测试文件" > uploads/image/test.txt
echo "测试图片" > uploads/image/test.jpg

# 编译后端
cd backend
go build -o star-cloud main.go
```

### 2. 检查目录结构

确保项目目录结构如下：
```
/www/wwwroot/axi-star-cloud/
├── backend/
│   ├── star-cloud (可执行文件)
│   └── ...
├── front/
│   └── ...
├── uploads/
│   ├── image/
│   │   ├── test.txt
│   │   └── test.jpg
│   ├── avatars/
│   ├── video/
│   ├── audio/
│   ├── document/
│   ├── md/
│   └── other/
└── index.html
```

### 3. 宝塔面板网站配置

#### 3.1 检查网站设置
1. 登录宝塔面板
2. 进入"网站"管理
3. 找到 `redamancy.com.cn` 网站
4. 点击"设置"

#### 3.2 配置反向代理
在"反向代理"中添加以下配置：

**代理名称**: `api`
**代理目录**: `/api`
**目标URL**: `http://127.0.0.1:8080/api`

**代理名称**: `uploads`
**代理目录**: `/uploads`
**目标URL**: `http://127.0.0.1:8080/uploads`

**代理名称**: `static`
**代理目录**: `/static`
**目标URL**: `http://127.0.0.1:8080/static`

#### 3.3 配置伪静态
在"伪静态"中添加以下规则：

```nginx
location /api {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location /uploads {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location /static {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

### 4. 服务管理

#### 4.1 创建系统服务
创建服务文件 `/etc/systemd/system/star-cloud.service`：

```ini
[Unit]
Description=Star Cloud Service
After=network.target

[Service]
Type=simple
User=www
WorkingDirectory=/www/wwwroot/axi-star-cloud/backend
ExecStart=/www/wwwroot/axi-star-cloud/backend/star-cloud
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

#### 4.2 启动服务
```bash
# 重新加载systemd
systemctl daemon-reload

# 启动服务
systemctl start star-cloud

# 设置开机自启
systemctl enable star-cloud

# 检查服务状态
systemctl status star-cloud
```

### 6. 常见问题排查

#### 6.1 端口占用
```bash
# 检查8080端口是否被占用
netstat -tlnp | grep 8080

# 如果被占用，杀死进程
kill -9 <进程ID>
```

#### 6.2 权限问题
```bash
# 检查uploads目录权限
ls -la /www/wwwroot/axi-star-cloud/uploads

# 重新设置权限
chmod -R 755 /www/wwwroot/axi-star-cloud/uploads
chown -R www:www /www/wwwroot/axi-star-cloud/uploads
```

#### 6.3 防火墙问题
```bash
# 检查防火墙规则
iptables -L

# 如果需要，开放8080端口
iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
```

#### 6.4 日志查看
```bash
# 查看服务日志
journalctl -u star-cloud -f

# 查看nginx日志
tail -f /www/wwwlogs/redamancy.com.cn.log
```

### 7. 自动化部署脚本

使用项目根目录的 `deploy.sh` 脚本：

```bash
# 给脚本执行权限
chmod +x /www/wwwroot/axi-star-cloud/deploy.sh

# 执行部署
cd /www/wwwroot/axi-star-cloud
./deploy.sh
```

## 总结

按照以上步骤配置后，uploads路径应该能够正常访问。如果仍有问题，请检查：

1. 后端服务是否正常运行
2. 反向代理配置是否正确
3. 文件权限是否正确
4. 防火墙是否阻止了访问
5. nginx配置是否正确

如有问题，请查看相关日志文件进行排查。
